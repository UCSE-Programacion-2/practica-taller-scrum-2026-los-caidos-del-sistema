# Sprint 0 — Relevamiento (Rent&Go / alquiler de vehículos)

## Objetivo

Documentar el **contexto** del negocio de alquiler, necesidades detectadas y material visual (referencias y logo) para que el equipo pueda diseñar el Sprint 1 con criterio.

Misma lógica que en el repo **demo docente** (`demo-proyecto-integrador`, Sprint 0 del integrador), pero el dominio es **alquiler de vehículos** según [docs/product-brief.md](../product-brief.md).

## Qué entregar en esta carpeta

1. `relevamiento.md`
   - Qué ofrece la empresa de alquiler, público, sucursales.
   - Problemas o necesidades (2 a 5), alineados al brief.
   - Qué impacta en la app (fechas, flota, reservas, pagos, etc.).

2. `fotos/` o `referencias-visuales/`
   - Mínimo 5 imágenes de **bancos con licencia** (sucursal genérica, vehículo de stock, mostrador).
   - En `relevamiento.md`, describir qué aporta cada grupo.

3. `logo/`
   - Propuesta de marca (SVG/PNG) con justificación breve de colores.

4. (Opcional) `extras/`
   - Paleta HEX, referencias de UX de otras apps de alquiler (sin copiar marca).
