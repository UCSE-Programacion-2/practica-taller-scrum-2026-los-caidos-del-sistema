# 00 - Preparación

## Objetivo del paso

Dejar listo el entorno para ejecutar el mini sprint sin bloqueos.

## Tiempo estimado

10-15 minutos.

## Acciones (paso a paso)

1. Si usan **GitHub Classroom**: las tres personas deben aceptar el **mismo link** de la assignment y **unirse al mismo equipo** (grupo de 3). Quedará **un solo repositorio** por equipo. Detalle: [github-classroom-y-template.md](./github-classroom-y-template.md).
2. Si **no** usan Classroom: crear un repo vacío e invitar a los otros dos integrantes como colaboradores.
3. Confirmar que los **tres** integrantes tienen acceso al mismo repositorio.
4. Verificar permisos para usar **GitHub Projects** en ese repositorio/organización.
5. Definir roles para el taller (hay 3 personas: repartir PO, SM y Dev; se puede rotar en otra sesión):
   - Product Owner (PO)
   - Scrum Master (SM)
   - Developer(s)
6. Completar el `README.md` raíz del repo con:
   - Nombre del equipo
   - Los **tres** integrantes con enlace a su perfil de GitHub
   - Link al tablero (se completará en el paso 01)
   - Si partieron de este repositorio como plantilla o desde Classroom, editar los placeholders ya existentes (incluidos los **tres** integrantes).

## Resultado esperado

- Los tres integrantes tienen acceso al mismo repositorio.
- Roles asignados para la sesión.
- README base visible en `main`.

## Si falla, revisar

- En Classroom: si cada uno tiene un **repo distinto**, probablemente crearon **equipos distintos**; deben usar el mismo nombre de equipo o invitación grupal según indique la cátedra.
- Permisos de colaborador en GitHub.
- Si Projects está deshabilitado en la organización.
- Que los tres estén usando la cuenta GitHub correcta.
