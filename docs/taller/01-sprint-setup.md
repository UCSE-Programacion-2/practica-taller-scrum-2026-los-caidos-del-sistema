# 01 - Sprint Setup en GitHub

## Objetivo del paso

Configurar el marco Scrum mínimo para trabajar de forma ordenada en la sesión.

## Tiempo estimado

20 minutos.

## Acciones (paso a paso)

1. Crear un GitHub Project (vista Board).
2. Configurar columnas:
   - Product Backlog
   - Sprint Backlog
   - In Progress
   - In Review
   - Done
3. Crear labels mínimos:
   - `epic`
   - `feature`
   - `task`
   - `bug`
   - `P1`, `P2`
4. Crear un milestone:
   - `Mini Sprint Taller Scrum`
   - Fecha de cierre: hoy (fin de la clase) o +1 día
5. Definir DoR (Definition of Ready) de taller:
   - Historia redactada en formato “Como / Quiero / Para”
   - Criterios de aceptación presentes
   - Esfuerzo pequeño (completable en clase)
6. Definir DoD (Definition of Done) de taller:
   - Tiene issue asociado
   - Tiene rama propia
   - Tiene PR con revisión (**otro** integrante del grupo de 3 aprueba; la cátedra define si 1 o 2 aprobaciones)
   - Está mergeado en `main`
7. Actualizar el `README.md` del repo con el link al tablero y comprobar que figuren los **tres** GitHub del equipo.

## Resultado esperado

- Tablero visible con columnas y labels.
- Milestone creado.
- DoR/DoD acordados por el equipo.

## Si falla, revisar

- El proyecto fue creado en otro repositorio por error.
- Falta vincular issues del repo al tablero.
- No se guardó el milestone con fecha.
